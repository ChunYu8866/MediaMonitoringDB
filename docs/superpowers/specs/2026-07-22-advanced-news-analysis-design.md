# 進階新聞分析工作台設計

## 目標

在既有免費新聞 Demo 新增類似專業輿情平台的核心分析工作台，同時修正來源時間被錯誤顯示為「剛剛」的問題。所有分析只使用既有 22 家新聞媒體，不加入社群或付費資料。

## 搜尋與比較

- 新增「進階分析」頁面，最多同時比較三個命名主題。
- 每個主題接受 AND、OR、NOT、減號排除詞與雙引號精準詞。
- Worker 與 Pages 靜態備援使用相同查詢語意，避免線上與備援結果不同。
- 每個主題分別呼叫現有即時搜尋 API；不建立新的付費服務或資料庫。

## 分析內容

- 主題總聲量與跨主題比較。
- 各主題聲量時間軸。
- 新聞來源分布。
- 實驗性正面、中立、負面比例；僅依標題與短摘要的公開字典計算並清楚標示限制。
- TOP 關聯詞與近期升溫詞；以同一搜尋區間前後半段的詞頻差計算，不宣稱全網趨勢。
- 文章列表可依主題、來源與情緒篩選，連回原文。

## 時間正確性

- RSS 日期無法解析時，捨棄該筆，不得以抓取時間冒充發布時間。
- Google News 若把台灣本地時間誤標為 GMT，造成時間落在未來，且減去八小時後合理，則校正為 UTC。
- UI 顯示絕對日期時間；相對時間僅用於已驗證不在未來的系統更新狀態。
- 新聞時間代表來源 RSS 提供時間；無法保證與原文頁二次編輯時間完全相同。

## 更新、錯誤與限制

- 沿用搜尋不共用快取與每 30 秒刷新。
- 任一主題失敗時保留其他主題結果並標示錯誤。
- 最多三個主題、每個查詢 2 至 50 字元，避免免費 Worker 被大量請求耗盡。
- 不加入登入、社群資料、不可追溯摘要或全文重製。

## 驗證

- Worker 測試涵蓋布林查詢、未來時間校正與無效日期捨棄。
- Web 測試涵蓋相同查詢語意、情緒分類與關聯詞統計。
- Python 快照解析器同步採用日期完整性規則。
- 正式部署後以已知三立文章驗證顯示時間不再多八小時，並驗證進階分析頁面資產與 API 結果。
